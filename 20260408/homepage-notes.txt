20260120 - I made a <a href="https://www.redactedchat.com">Redacted Science chatbot</a> prototype locally, then I deployed it to the web! This uses about 5 or 6 technologies I've never touched before and it only took 6-8 hours. Redacted Chat is built on a retrieval-augmented generation (RAG) architecture. Book content from Redacted Science is stored in a Supabase PostgreSQL database with vector embeddings generated using OpenAI’s embedding models. When a question is asked, the system performs a semantic vector search to retrieve the most relevant passages, which are then passed—along with the user’s question—to Anthropic’s Claude model to generate an answer constrained to the source material. A lightweight Python FastAPI service hosted on Render orchestrates retrieval and generation while securely managing API keys. The browser-based chat interface is served as a static site via Vercel, with DNS and routing handled through my hosting service, cleanly separating the public frontend from the backend API.
